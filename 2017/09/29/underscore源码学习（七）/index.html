Sep 29, 2017

节流函数

我们平时在写代码的时候，肯定会有这种需求，当你提交一个ajax请求时，当这个请求还没返回响应的时候，用户可能会等不及，而进行多次点击，从而发送了多次请求。显然，这并不是我们所想看到的，一方面，用户体验不好，另一方面，对服务器造成了负担。一般我的解决方法是，给这次请求设置一个loading变量，默认值为false，在发送请求之前设置loading = true，在得到请求相应之后设置loading = false。如果用户在第一次请求未返回响应的时候多次点击提交ajax请求，那么此时的loading的值是true，即当前请求还没处理完，用条件语句控制一下if(loading){}，使其请求不能发送， 同时给出对应的界面提示。

但是这种方法并不能很好的处理，解决更复杂的需求，所以更多的时候我们采用的是用节流函数来解决。看了underscore中的节流函数，我发现我之前写过的节流函数是真的菜，underscore写的更加全面完善，更加宽松。

_.throttle

_.throttle翻译过来有节流阀的意思， 很形象（值得我们学习的还有作者的参数以及函数命名方式）。通过_.throttle函数可以限制函数的执行频次。本质上这个函数还是通过setTimeout来实现间隔调用的，看下它的实现

一开始看_.throttle函数，还不能很快的搞清楚他的执行过程，但是多看几次并结合官方文档例子，慢慢的就搞清楚了。_.throttle通过判断remaining和wait的差别判断函数是否立即执行。在执行func之后同时刷新了previous的值，以便下次判断使用，如果用户同时多次点击了，并不会重复执行，因为在满足remaining条件后，_.throttle会清空timeout，也就是重复点击的事件不会存在，而是被清除了，所以就不会存在多个回调函数在同一时间点执行这种情况了。
_.throttle函数还支持options配置，参数分别是

  • leading: Boolean
  • tailing: Boolean

leading的作用是保证第一次调用可以被立即执行，如果传入leading为false,则第一次调用也要等待wait后才能执行，默认值为true
tailing的作用是当你最近一次尝试调用func时，如果func不能被执行，那么func将被延时执行，默认为true

_.debouce

_.debouce函数基于_.throttle的思路实现，返回func的防反跳版本。将延迟函数的执行(真正的执行)在函数最后一次调用时刻的 wait 毫秒之后. 对于必须在一些输入（多是一些用户操作）停止到达之后执行的行为有帮助。 例如: 渲染一个Markdown格式的评论预览, 当窗口停止改变大小之后重新计算布局, 等等。

当immediate为true时，_.debounce会在 wait 时间间隔的开始调用这个函数 。

underscore中提供了一个函数组合的函数，类似管道流，即把当前函数执行的结果作为参数传递给下一个函数，依次执行，直到所有函数执行完，返回最后的结果。
