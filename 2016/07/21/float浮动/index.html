float浮动

CSS
  • CSS

前言

算算时间，学习html和css也有一段时间了，打算整理整理浮动这个技术，一开始学习浮动的时候遇到很多坑，例如，一个块浮动之后，其他块的位置会怎么变化，当一个父元素的子元素浮动之后，父元素的高度为什么会塌陷，浮动之后脱离文档流等等。在翻看大牛的博客和网上的资料之后，对浮动有了一定的理解。

文档流

所谓的文档流实际上是一种民间说法，正常称谓应该指的是常规流（normal flow），个人感觉这两者应该是“阀值”与“阈值”的关系。从直观上理解正常流指的是元素按照其在 HTML中的位置顺序决定排布的过程，主要的形式是自上而下，一行接一行，每一行从左至右。

  1. 常规流，也就是俗称的文档流，又包含三种类型：块级元素的块级格式、行内元素的行内格式以及两种元素的相对定位方式。
  2. 浮动（Float）
  3. 绝对定位（Absolute Positioning）：包含 absulote 和 fix 两种

所谓的不脱离文档流就只有三种情况：块级、行内和相对定位。由此上面遇到的第一个问题也就得以证实：相对定位是不脱离文档流的。

脱离文档流的问题也迎刃而解，就只有两种情况：浮动 和 绝对定位。而这两者的表现实际上非常相似，区别仅在于宽度缺失

浮动

浮动可以让页面元素并排显示，而且还可以制作一个干净的布局。然而，有时候需要一些严格的定位，这时需要使用其他的技术，包括“relative”和“absolute”定位。创建一全页面的布局时，浮动是一种常用的方法，也是页面元素定位的一种基本功能。浮动可以让元素一个挨着一个。浮动可以创建一个自然流布局，同时充许元素设置自身尺寸和其父元素容器的尺寸大小。

浮动问题-父元素高度塌陷

一个父元素包含了多个浮动的子元素。页面的内容设置了一个宽度，子元素的浮动确定了他们的位置，但浮动元素不会影响父元素的宽度。这样做会让父元素塌陷，从而使父元素的高度为“0”，以及忽略其他的属性。很多时候，这种现像都被忽略，特别是在父元素没有任何样式，以及其子元素看起来都正确的对齐。如下例子所示，在“.parent”div下应该有一个黑色的背景，但在其子元素浮动之后，这个背景却看不到了，打开盒子模型之后才发现，“.parent”的高度变为0。

清除浮动

clear:both

有一种方法，在容器的结束标签前添加一个空标签，在空标签上直接设置样式“clear:both”。用这种方法来清除浮动，在大多数情况下是有效的，但这不太适合语义化。这取决于一个页面有多少浮动需要清除，这样造成页面上的空标签迅速堆积，而且在页面中没有上下文内容。

overflow技巧

使用“overflow”属性。在具有浮动元素的父容器中设置“overflow”的属性值为“auto”，这样父容器就会有一个高度存在，例子中的黑色背景就能看到了。

clearfix技巧

另外一种清除浮动的技巧是在最后一个子元素后面添加一个class=“clearfix”的div，使用伪类清除浮动，代码如下：

.clearfix:after {
    content: ".";
    height: 0;
    clear: both;
    visibility: hidden;
    display: block;
}

有效的包含浮动

使用哪种技巧来清除浮动，终究要看你自己喜好。有些人坚持使用“clearfix”来清除浮动，因为这种方法可以贯穿整个项目。有些人认为“clearfix”技巧使用的代码太多，他还是喜欢简单点的。至于使用什么技巧由您来决定，只要在运用了浮动的元素的父容器需要清除浮动。个人最常使用的是第三种方法。
