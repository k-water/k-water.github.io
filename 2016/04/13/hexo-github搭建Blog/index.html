hexo+github搭建Blog

Apr 13, 2016

前言

为什么要搭建一个独立博客?

独立的才是自己的。我想到些博客也是缘于自己的兴趣，在写博客的过程中，一方面是给自己做笔记，可以提升自己的写作、总结能力，一个技术点我们会使用，并不难，但是要做到让让别人也能听懂我们讲得，还是需要一定的技巧和经验的。写博客是一个不错的锻炼机会。

喜欢写Blog的人，会经历三个阶段。

引用自阮一峰老师

  1. 第一阶段，刚接触Blog，觉得很新鲜，试着选择一个免费空间来写。
  2. 第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
  3. 第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

Hexo简介

hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github和Heroku上。作者是来自台湾的@tommy351。

我为什么使用Hexo

引用作者的原话

不可思议的快速 ─ 只要一眨眼静态文件即生成完成
支持 Markdown
仅需一道指令即可部署到 GitHub Pages 和 Heroku
已移植 Octopress 插件
高扩展性、自订性
兼容于 Windows, Mac & Linux

开始搭建属于自己的博客

安装Git

很好用的工具，把本地的hexo内容提交到github上，请到官网自行下载。安装完成后在桌面右键，点击Git Bash即可使用

Github账号

通过官网注册一个账户，这个就不多说了。

接下来使用github为每个用户使用的二级域名，创建一个属于自己的仓库，可以时时预览自己上传的网页。如下图，其中仓库的名字格式为：username.github.io(只有这样才能生效)。设置完成后再浏览器中键入username.githu.io测试，就可以了。

安装Node.js

这是比不可少的一步，从官网上下载一路安装就可以，安装成功后，打开cmd（win+R），测试是否安装成功。下图为本地测试结果

安装Hexo

通过命令行 npm install -g hexo安装。PS：打开cmd输入此命令等待安装完成即可

安装完成后，初始化本地博客目录，选择你想存储的位置。例如，我在F盘下创建了一个blog文件夹，即F:/blog用来存放本地博客。进入此文件夹，右键使用git bash。输入hexo init回车。
好啦，至此，全部安装工作已经完成！blog就是你的博客根目录，所有的操作都在里面进行。

紧接着，cd 到你的init目录，执行hexo generate（hexo g也可以）命令，生成静态页面至blog\public\目录。
命令必须在init目录下执行，否则不成功，但是也不报错。当你修改文章Tag或内容,不能正确重新生成内容，可以删除hexo\db.json后重试，还不行就到public目录删除对应的文件，重新生成。

本地测试，输入hexo server如下图,

然后在浏览器输入localhost:4000，请使用高级浏览器，否则呵呵哒，你懂的。至此，本地搭建基本完成。

若使用ssh密钥，则每次部署不需要输入账户和密码，_config.yml也要做相应的修改。

部署步骤

每次部署的三个步骤
hexo clean
hexo generate
hexo deploy

常用命令

hexo new"postName" #新建文章
hexo new page"pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本
